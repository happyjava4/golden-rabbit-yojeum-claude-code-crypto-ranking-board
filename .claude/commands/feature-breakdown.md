# 🧩 문제 세분화 커맨드

## 입력

- 이슈: $ARGUMENT

## 목표

- 이슈를 작은 작업 단위로 나누어 실행 가능한 작업 목록을 만든다.

## 출력 형식

- 작업 목록 (우선순위 포함)
- 각 작업의 목표/입력/출력
- 의존성 및 선행 조건
- 검증 방법

---

## 분해 지침

1. 최종 목표를 한 문장으로 요약한다.
2. 기능/흐름/데이터 관점 중 하나를 선택해 분해한다.
3. 각 하위 작업은 1~3시간 내 수행 가능한 크기로 만든다.
4. 의존성이 있는 작업은 순서를 명시한다.
5. 각 작업에 성공 기준과 검증 방법을 붙인다.
