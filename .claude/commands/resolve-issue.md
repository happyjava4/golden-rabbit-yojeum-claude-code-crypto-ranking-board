# 🧭 GitHub 이슈 기반 작업 계획 프롬프트

## 입력

- 이슈 번호: $ARGUMENT

## 목표

- GitHub에서 이슈 내용을 가져와 핵심 요구사항을 정리하고,
- 해결을 위한 상세 실행 계획을 작성한 뒤,
- 사용자에게 승인(검토)을 요청한다.

---

## 수행 절차

1. GitHub에서 이슈 #$ARGUMENT의 제목/본문/라벨/댓글 요약을 불러온다.
2. 요구사항을 핵심 항목으로 정리한다.
3. 해결 계획을 단계별로 작성한다.
4. 위험 요소/불확실성을 명시한다.
5. 사용자에게 검토 및 승인 요청을 한다.

---

### 1) 이슈 요약
