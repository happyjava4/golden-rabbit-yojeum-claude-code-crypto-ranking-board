# 🧩 도시 상세 페이지 작업 분해

## 목표

- 도시 카드를 눌렀을 때 상세 페이지로 이동하는 기능을 구현하기 위한 작업을 실행 가능한 단위로 분해한다.

---

## 작업 분해

### 1) 라우팅 설계

- 상세 페이지 URL 규칙 정의 (예: `/cities/[id]` 또는 `/cities/[slug]`)
- 라우트 파라미터 타입 결정 (`id` 숫자 vs `slug` 문자열)

### 2) 데이터 연결

- 도시 데이터 조회 함수 준비 (예: `getCityById`, `getCityBySlug`)
- 상세 페이지에서 파라미터 → 도시 데이터 매핑
- 도시가 없을 때 처리(404/NotFound)

### 3) 카드 → 상세 페이지 이동

- `CityCard` 클릭 영역 결정 (전체 카드 vs 버튼)
- 클릭 시 해당 도시의 상세 경로로 이동하도록 링크 추가

### 4) 상세 페이지 UI 구성

- 핵심 정보 영역: 이름, 지역, 태그, ASCII 아트
- 상세 정보 영역: 예산, 환경, 추천 계절 등
- 반응/지표 영역: 좋아요/싫어요 등

### 5) 접근성/UX 보완

- 뒤로가기 링크 제공
- 모바일 레이아웃 대응 (그리드/스택 전환)
- 비어있는 데이터에 대한 기본 값 처리

### 6) 테스트/검증

- 상세 페이지 렌더링 테스트 (도시 존재 시)
- 잘못된 파라미터 처리 테스트 (도시 없음)
- 카드 클릭 시 링크 경로 테스트
